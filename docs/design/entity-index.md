# NVA Pet Resorts entity index

This page helps a non-coder pick an important pet-resort entity and quickly answer: what is it, why does it reduce labor, which workflows feature it, what it relates to, who or what is authoritative, what automation may draft or recommend, what remains human-reviewed or blocked, how value is measured, and where source/Rustdoc/test evidence lives.

The root README maxim is preserved here: this repo is an entity-first operating model for labor-cost reduction. The index intentionally avoids a crate-name-first organization. Crate/module names appear only as evidence links after the pet-resort meaning is clear. When this page uses architecture terms, read them in their operator sense: [contract](../glossary-architecture-terms.md#contract) means a source-backed rule/promise, [semantic](../glossary-architecture-terms.md#semantic) means validated business meaning, [projection](../glossary-architecture-terms.md#projection) means reporting/database view rather than live authority, and [promotion/demotion](../glossary-architecture-terms.md#promotion-demotion) means explicit conversion between boundary data and trusted business meaning.

## How to use this index

1. Start with the entity name or family in the tables below.
2. Use the family page link for a fuller plain-English explanation and relationship map.
3. Use source/Rustdoc/test paths as evidence, not as the first reading path.
4. Treat Markdown as navigation. The behavior is authoritative only when backed by source contracts, Rustdoc paths, tests, fixtures, or clearly named human/system-of-record authority.

Default automation boundary across this index:

- Allowed: read source-backed evidence, validate deterministic gates, summarize, rank, draft, route, create audit drafts, preserve ambiguity, and record reviewed outcomes.
- Blocked unless a separate app-owned approval contract and human/system-of-record action exists: customer/member sends, Gingr/PMS/provider writes, booking/check-in/checkout/schedule/capacity changes, payment/refund/discount movement, medical/vaccine/behavior/safety approval, policy exceptions, source hiding, and sensitive payload exposure.

## Relationship shortcut

For any entity above, the safe flow is:

```text
provider/staff/import evidence
  -> source ref / provenance / data-quality status
  -> domain entity or service-line truth
  -> app workflow packet / review queue
  -> optional agent draft or summary
  -> human/system-of-record review
  -> outcome record + labor minutes
  -> storage/API/runtime proof
```

Use [Entity atlas relationship map](entity-atlas-relationships.md) for the full edge table, including Gingr/provider mapping, service-line workflow edges, blocked-action families, and family-to-family adjacency.
