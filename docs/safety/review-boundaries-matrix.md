# Review boundaries matrix for agent workflows

Glossary bridge: this matrix should link only the terms that define authority boundaries: [review gate](../glossary-workflow-state-terms.md#review-gate), [blocked action](../glossary-workflow-state-terms.md#blocked-action), [draft](../glossary-workflow-state-terms.md#draft), [tool port](../glossary-architecture-terms.md#tool-port-apptools), and [source-of-record](../glossary-source-data-terms.md#source-of-record).

Purpose: turn the source-backed safety map in `docs/safety/source-evidence-map.md` into operational review lanes for booking triage, manager daily brief, daily care update/customer messaging, checkout/retention, tool ports, provider portal, payment, document/vaccine/medical evidence, and Hermes automation hooks. This is a routing aid for operations, IT/security, product, and compliance readers; it is not a legal/compliance guarantee and it does not approve production live-write authority.

Use this matrix with the current source anchors from the evidence map:

- Agent packet/spec contract: `app/src/agents.rs`.
- Booking triage: `app/src/booking_triage.rs` (`app::booking_triage`).
- Manager daily brief: `app/src/manager_daily_brief.rs` (`app::manager_daily_brief`) and `domain/src/daily_brief.rs`.
- Tool ports: `app/src/tools.rs` and `app/src/tools/error.rs`.
- Policy/workflow/source vocabulary: `domain/src/policy.rs`, `domain/src/workflow.rs`, and `domain/src/source.rs` (`domain::policy`, `domain::workflow`, `domain::source`).
- Outcome capture: `storage/src/operations.rs`, especially `ManagerDailyBriefOutcomeRecord` and `DataQualityHygieneOutcomeRecord`.

## How to read the reviewer lanes

- Staff review means a front-desk or care-team operator checks the packet against source records and chooses the next operational step. Staff can own routine intake, missing-info, daily-care, checkout, and internal task decisions when no manager, medical, payment, or customer-message escalation is present.
- Manager review means the action affects capacity, staffing, policy exceptions, hard stops, incidents, schedule/capacity risk, or customer-trust decisions. The manager lane owns approvals that change how the resort operates, not just whether a draft is well written.
- Medical/vaccine qualified staff review means a person authorized by the resort checks vaccine proof, medical/care-document evidence, medication/special-care ambiguity, or behavior/safety eligibility. OCR text and agent summaries are evidence only.
- Customer-message review means a human approves final text, recipient, channel, timing, and whether the message should be sent at all. Low-risk drafts can be queued for review; sensitive messages stay draft-only or never-auto-send unless a later deterministic path is approved.
- Payment/accounting review means a human or approved payment workflow checks deposits, refunds, waivers, discounts, balances, and provider receipts. Agent output can summarize and draft packets, but it cannot move money.
- Provider portal review means IT/security and product/ops decide whether any integration may write back to a provider system. Current tool ports describe read, lookup, draft, and failure contracts; they do not grant live provider mutation authority.

Tool ports are boundaries, not permissions. `app/src/tools.rs` names narrow app-owned adapters such as customer-store reads, availability checks, draft reservation updates, portal lookups, payment helpers, message drafting, document/OCR helpers, media snapshots, and Hermes task/schedule drafts. These ports describe what an integration can return and how failures become review work. They do not let an agent bypass `domain::policy` review gates, `domain::workflow` validation, source provenance in `domain::source`, or human approval for live customer/provider/payment actions.

## Blocked actions that should become review packets or blocked tickets

Create a review packet or blocked ticket instead of an autonomous write when any of these appear:

- Booking triage wants to confirm, reject, cancel, waitlist-release, check in/out, allocate capacity, or promise availability before staff/manager approval.
- A draft message contains health, medication, allergy, behavior, incident, safety, legal, payment, refund, eligibility, refusal, complaint, bad-review, policy-exception, or ambiguous facts.
- OCR/document extraction is low-confidence, dates or pet identity are ambiguous, licensed-vet proof is unclear, or vaccine/medical evidence affects check-in or eligibility.
- A payment path involves capture, refund, void, retry, waiver, discount, forfeiture, write-off, deposit exception, duplicate risk, amount mismatch, provider ambiguity, or customer action.
- Manager daily brief output would change a staff schedule, hide a source data-quality problem, move money, mutate provider/PMS records, or send a customer follow-up.
- A provider portal lookup is ambiguous, not found, conflicts with local records, or requires a merge/create/update in the provider system.
- A tool port returns an external failure, permission denied, camera/media unavailable, stale source snapshot, unmapped provider value, or unsupported side effect.
- A Hermes automation hook proposes a schedule change, recurring automation, webhook, or integration behavior that changes live operations rather than drafting internal work.

The safe default is: preserve source refs, explain the requested action, name the required reviewer lane, and record the outcome only after that reviewer approves, rejects, defers, or corrects the packet.
