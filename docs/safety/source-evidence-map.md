# Source evidence map for agent safety and human review

Glossary bridge: source evidence terms should point readers to [Gingr/source](../glossary-source-data-terms.md#domainsourcesystemgingr-gingr), [provider record](../glossary-source-data-terms.md#provider-record), [source-of-record](../glossary-source-data-terms.md#source-of-record), [source refs/provenance](../glossary-source-data-terms.md#domainsourceprovenance-and-domainsourcerecordref-as-data-evidence), [data-quality issue](../glossary-source-data-terms.md#domaindata_qualityissue-data-quality-issue), and [storage projection](../glossary-architecture-terms.md#storage) before claiming source authority.

Purpose: give non-coder documentation writers a source-backed map for safety, human-review, and audit claims. This file is a citation map, not a new product contract. If a capability is not named in the cited source or docs, do not describe it as available.

Use these paths as the primary citation surface for later operations, IT, compliance, and product docs:

- Workspace and glossary: `README.md`.
- App workflow wiki: `app/README.md`.
- Shared agent packet/spec contract: `app/src/agents.rs`.
- Booking triage contract: `app/src/booking_triage.rs`.
- Manager brief workflow: `app/src/manager_daily_brief.rs` and `domain/src/daily_brief.rs`.
- Tool-port boundaries: `app/src/tools.rs` and `app/src/tools/error.rs`.
- Domain safety vocabulary: `domain/src/workflow.rs`, `domain/src/policy.rs`, and `domain/src/source.rs`.
- Persistence/audit-friendly records: `storage/src/operations.rs`.
- Labor-loop docs: `docs/design/labor-cost-reduction-crosswalk.md`, `docs/design/manager-daily-brief-measurable-labor-loop.md`, and `docs/design/data-quality-hygiene-labor-loop.md`.
