# Docs usefulness/pruning Kanban board

Purpose: make the NVA repo docs not merely green, but useful as a labor-cost-reduction operating model. The work should improve reader wayfinding, flow/chain/contract comprehension, canonical-vs-background boundaries, and evidence navigation without creating more generic documentation volume.

Strategic acceptance lens:

- The repo should read as a source-grounded, review-gated workflow automation platform for lowering pet-resort labor cost.
- Gingr/provider systems are source evidence, not the strategy.
- BI/read models/reporting DBs are visibility and measurement surfaces, not workflow authority.
- Canonical docs should make it easy to follow: source evidence -> provenance/data quality -> domain entity/contract -> app workflow packet -> agent-safe draft/ranking/summary -> human review gate -> outcome/labor minutes -> storage/runtime/reporting proof.
- Older planning/audit/kanban/discovery docs must not compete with canonical reader paths.

Initial board graph:

1. Create/polish canonical docs index and README wayfinding.
2. Normalize `nva-pet-resorts-ai-context.md` as background/discovery and link it forward.
3. Turn `docs/design/labor-cost-reduction-crosswalk.md` into a better navigation hub.
4. Add one concrete end-to-end flow walkthrough, likely Manager Daily Brief or Data Quality Hygiene.
5. Add visible successor/archive mapping for older docs and workflow specs.
6. Reduce duplicated safety prose by pointing to canonical safety/authority surfaces while preserving workflow-specific blocked actions.
7. Final independent review: non-coder usefulness, link health, docs checks, and proof-chain readability.
